投稿の内容をカードのデザインで表示し、一覧画面でも流用できるように実装

カードのデザインについて（https://daisyui.com/components/card/）

## `<div class="max-w-3xl mx-auto">`

**`max-w-3xl`** → 横幅を制限（広がりすぎ防止）

**`mx-auto`** → 中央寄せ

## `<div class="card bg-base-100 shadow-xl border border-base-200">`(カード全体のデザイン)

**`card`** → DaisyUIのカードUI

**`bg-base-100`** → 背景色（白っぽい）

**`shadow-xl`** → 影（浮き上がる感じ）

**`border`** → 枠線

## `<div class="card-body">`(カードの中身)

**内側の余白（padding）を自動でつける**

## `<h2 class="card-title text-2xl">`

**`card-title`** → カードの見出し用スタイル

**`text-2xl`** → 文字大きく

## `<div class="divider my-1"></div>`

**`divider`** → 横線

**`my-1`** → 上下の余白

## `<div class="grid gap-3 sm:grid-cols-2">`

**`grid`** → グリッドレイアウト

**`gap-3`** → 要素間の余白

**`sm:grid-cols-2`** → スマホで縦1列、PCで2列

## `<span class="font-bold">プラットフォーム：</span>`(ラベル)

**`font-bold`** → 太字

## `str_repeat('★', $game->rating)`(評価を星で表示)

```
4 → ★★★★☆
```

参考サイト（https://www.php.net/manual/ja/function.str-repeat.php）

## `<span class="badge badge-success">`(ステータスをバッジで表示)

**`badge`** → ラベルUI

**`badge-success`** → 緑

## `intdiv(...), %`

プレイ時間を時間と分で表示

## `<div class="bg-base-200 rounded-lg p-4 whitespace-pre-wrap">`(レビュー表示)

**`bg-base-200`** → 少しグレー背景

**`rounded-lg`** → 角丸

**`p-4`** → 内側余白

**`whitespace-pre-wrap`** → 改行を保持

## ボタン

### `<div class="card-actions justify-end mt-6">`

**`card-actions`** → ボタン用

**`justify-end`** → 右寄せ

**`mt-6`** → 上に余白

### `<a class="btn btn-outline">`

**`btn`** → ボタン化

**`btn-outline`** → 枠だけ

参考になるサイト（https://qiita.com/morrrrr/items/d0dfa70ede3165d01b36）

## 編集ボタン部分
```
<a href="{{ route('games.edit', $game) }}" class="btn btn-outline btn-success">編集</a>
```

### ① `route('games.edit', $game)`

**意味：編集画面へのURLを生成**

### ② `<a href="...">編集</a>`

普通のリンク

### ③ `class="btn btn-outline btn-success"`

- btn → ボタン化
- btn-outline → 枠だけ
- btn-success → 緑

## 削除ボタン
```
<form action="{{ route('games.destroy', $game) }}" method="POST">
```

### ① `route('games.destroy', $game)`

削除用URL

### ② `method="POST"`

HTMLは DELETE が使えないので、POSTで送る

### ③ `@csrf`

CSRF対策、不正なリクエスト防止

**セキュリティ用トークン**

これがないと、419エラーになる

csrfについて（https://www.ipa.go.jp/security/vuln/websecurity/csrf.html）

CSRF（Cross Site Request Forgery）<br>
サイトを跨いだなりすまし攻撃

「このフォームは本当にこのサイトから送られたか？」を確認することにより、対策してるシステム

### ④ `@method('DELETE')`

疑似的にDELETEリクエストにする

実際は、`POST + _method=DELETE`

Laravelが解釈して、DELETEとして処理

### ⑤ `<button type="submit">`

フォーム送信

### ⑥ `onclick="return confirm('この記録を削除しますか？')"`
```
確認ダイアログを出す
↓
OK → true → 送信
キャンセル → false → 中断
```

### ⑦ `class="btn btn-outline btn-error"`

btn-error → 赤

### ⑧ `style="display:inline;"`

formは通常、ブロック要素（改行される）なので

- 編集ボタン
- 削除ボタン

を横並びにするため
