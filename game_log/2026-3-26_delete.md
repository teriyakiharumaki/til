# 投稿の削除機能の実装

## ✅ 行ったこと

- 一覧画面から投稿を削除する機能の実装について

## ①ルーティング
routes/web.phpに追記

```
Route::delete('/games/{game}', [GameController::class, 'destroy'])->name('games.destroy');
```

### `Route::delete(...)`
**deleteリクエスト**を受け取るルート

### `/games/{game}`
URLのパス

**`{game}ってなんだ`**：URLの中の変数

投稿ごとのIDが入る

### `[GameController::class, 'destroy']`

**GameController の destroy メソッドを呼ぶ**

### `->name('games.destroy')`
ルートに名前をつけている、Bradeで
```
route('games.destroy', $game)
```
と書けるようになる

## ②コントローラーにdestroyを追記
game_log/app/Http/Controllers/GameController.php

```
public function destroy(Game $game)
{
    $game->delete();

    return redirect()->route('games.index');
}
```

***

```
public function destroy(Game $game)
```
削除処理用のメソッドを定義

```
Game $game
```
これは、Laravelの Route Model Binding という仕組み
```
/games/3
```
にアクセスされたら、3番のGameをDBから探して、$game に自動で入れてくれる

つまり自分で
```
$game = Game::find($id);
```
って書かなくてよくなる

***

```
$game->delete();
```
これが削除本体

意味は**このゲームのレコードをデータベースから削除する**

たとえば id=3 のゲームだったら、gamesテーブルの3番のデータが消える

***

```
return redirect()->route('games.index');
```
削除した後に、`/game`の一覧ページに戻す

## ③indexに削除ボタンの追加
game_log/resources/views/games/index.blade.phpの各投稿の中に記述

```
<form action="{{ route('games.destroy', $game) }}" method="POST" style="display:inline;">
  @csrf
  @method('DELETE')
  <button type="submit" onclick="return confirm('削除しますか？')">削除</button>
</form>
```

***

```
<form ...>
```
削除ボタンは実は form で送信

LaravelでDELETEリクエストを送るには form を使う

***

```
action="{{ route('games.destroy', $game) }}"
```
formの送信先を指定

たとえば `$game->id = 3` なら
```
/games/3
```
になる

つまり、**このゲームを削除するURLへ送る**

***

```
method="POST"
```

HTMLのformは本来

- GET
- POST

しか送れない

だから見た目はPOSTで送って、Laravel側で「これはDELETEとして扱って」と変換する。（それが `@method('DELETE')`）

***

```
style="display:inline;"
```

formは普通だとブロック要素なので、改行されやすい

これを付けると、

- 他の文字の横に並べやすい
- 一覧の中で自然に削除ボタンを置ける

***

```
@csrf
```

Laravelでフォーム送信するときの **セキュリティ対策**

詳しくは、(2026-3-22_create_view.md)

***

```
@method('DELETE')
```

HTMLでは DELETE を直接送れないから、

**このPOST送信を、LaravelではDELETEリクエストとして扱ってね**

と伝えている。

これがあるから、さっきの
```
Route::delete(...)
```
にちゃんと届く

***

```
<button type="submit">削除</button>
```
押したらフォーム送信されるボタン

***

```
onclick="return confirm('削除しますか？')"
```

**削除前に確認ダイアログを出している**

押すと

- OK → 削除実行
- キャンセル → 削除しない

```
return confirm(...)
```

- true なら送信する
- false なら送信しない

だから、確認付き削除になる
