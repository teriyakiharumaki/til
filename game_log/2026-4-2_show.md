## ✅ 行ったこと

- 投稿の詳細画面の実装について

## ①ルーティング
game_log/routes/web.php
```
Route::get('/games/{game}', [GameController::class, 'show'])->name('games.show');
```

### `Route::get(...)`

**GETリクエストのときに動くルート**

GETは、ブラウザで普通にページを開くときのリクエスト

- 一覧画面を開く
- 詳細画面を開く
- フォーム画面を開く

### `'/games/{game}'`

URLパス

**「どのゲームを表示するか」をURLから受け取る**

`{game}`は、2026-3-26_delete.mdを参照

### `[GameController::class, 'show']`

**GameController の show メソッドを呼ぶ**

### `->name('games.show')`
ルートに名前をつけている、Bradeで
```
route('games.show', $game)
```
と書けるようになる

## ②コントローラーにshowを追記
game_log/app/Http/Controllers/GameController.php
```
public function show(Game $game)
{
    return view('games.show', compact('game'));
}
```

### `public function show(...)`

show という名前のメソッド

ルーティングの
```
Route::get('/games/{game}', [GameController::class, 'show']);
```
と対応してる

つまり、**/games/1 にアクセスしたとき、この show() が呼ばれる**

### `Game $game`

ここは Route Model Binding によって、

/games/1

なら

Laravel が自動でDBから1番のGameを探してくれて、その結果を $game に渡している。

つまり $game の中には、

```
title
platform
rating
review
play_time_minutes
```
みたいな、1件分のゲームデータが入ってる。

Route Model Bindingについて（https://zenn.dev/taroosg/articles/20240925175812-85e66e02a13362）

### `return view(...)`

**games/show.blade.php を表示する**<br>
**そのとき、$game という変数をViewに渡す**

という意味

### `compact('game')` って何？

これは PHP の関数で、以下のような意味
```
['game' => $game]
```
つまり、Controllerの中の $game を、Viewでも $game として使えるようにしてる

## ③viewの実装


### routeの中の`$game`の役割

ルートは以下だった
```
Route::get('/games/{game}', [GameController::class, 'show'])->name('games.show');
```
ここに {game} がある。

つまりURLは、/games/1 となる

この「1」が必要

#### `$game` を渡すと何が起きる？
```
route('games.show', $game)
```

Laravelは裏で`$game->id` を取り出して、 `/games/1` というURLを生成する

Route Model Bindingによって、`$game` を渡すだけで `/games/{id}` に変換してくれる
